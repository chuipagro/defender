# Code Style

## Formatting — clang-format

Style: **Allman**, 4 spaces indentation, 100 columns max.

```bash
# Format all sources in-place
clang-format -i src/**/*.cpp include/**/*.hpp
```

---

## Linting — clang-tidy

Requires a `compile_commands.json` (generated by CMake).

```bash
# Generate compile commands first
cmake -S . -B build -DCMAKE_EXPORT_COMPILE_COMMANDS=ON

# Run linter
clang-tidy -p build src/**/*.cpp
```

---

## Commit Convention

This project follows [Conventional Commits](https://www.conventionalcommits.org/):

```
<type>: <short description>

feat: add basic tower placement
fix: prevent enemies from clipping through walls
perf: cache path computation in WaveManager
refactor: extract EventBus to header-only template
docs: add architecture diagrams
chore: update CI to Ubuntu 22.04
```

!!! note "Why Conventional Commits?"
    PR titles are parsed by the `release-drafter` workflow to auto-generate categorized release notes and apply labels automatically.
